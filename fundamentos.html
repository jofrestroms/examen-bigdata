FUNDAMENTOS BIG DATA — 5V
EV2 — PIPELINE BATCH (NYC TLC TAXI)
Volumen: Dataset de 387.7 MB en CSV (millones de registros de viajes individuales), excediendo la capacidad de herramientas ofimaticas tradicionales y justificando Cloud Storage + BigQuery.

Velocidad: Pipeline Batch orquestado por Cloud Scheduler con ingesta periodica programada. Procesamiento por lotes del dataset completo de NYC Yellow Taxi Trip Data (Enero 2026).

Variedad: Datos en formato Parquet, materializados como CSV tabular con tipos heterogeneos: texto (VendorID), temporales (tpep_pickup_datetime, tpep_dropoff_datetime), numericas (trip_distance, fare_amount, tip_amount, tolls_amount, total_amount).

Veracidad: ETL con reglas estrictas: eliminacion de registros con trip_distance ≤ 0, total_amount ≤ 0, valores nulos en payment_type y RatecodeID, deduplicacion exacta, y filtro de anomalias temporales.

Valor: Dashboard en Power BI con conectado a BigQuery: distribucion horaria, zonas de alta demanda, analisis de tarifas, KPI operativos (distancia promedio, ingresos por dia, propinas por zona).
EV3 — PIPELINE STREAMING (SUBASTA COMPONENTES)
Volumen: Sucesion continua de eventos individuales (~5,600 registros) cuya acumulacion justifica el almacenamiento escalable en Cloud Storage.

Velocidad: Pipeline streaming con ventanas de 12 segundos en Dataflow, latencia de ~37 segundos (data lag) y ~12 minutos (disponibilidad final). Contraste directo con el modelo Batch de EV2 (horas).

Variedad: Datos ingresan como JSON via webhook HTTP POST y se materializan como CSV con campos de texto (cliente, producto, forma_pago), temporales (fecreg) y numericos (precio, cantidad, monto).

Veracidad: Diez campos estandarizados (id_cliente, cliente, genero, id_producto, producto, precio, cantidad, monto, forma_pago, fecreg). Limpieza via Alteryx/Dataprep.

Valor: Dashboard en Looker Studio: distribucion por genero (63.2% M / 36.8% H), volumen por empresa (Meta, Salesforce, Alphabet), montos por producto.

ANALISIS DE DATOS
EV2 — NYC TLC TAXI
Dataset: NYC Yellow Taxi Trip Data (Enero 2026) — millones de registros procesados via pipeline Batch.

Hallazgos clave del dashboard en Power BI:
  • Distribucion horaria: Identificacion de horas punta y valles de demanda segun volumen de viajes por hora del dia.
  • Zonas de alta demanda: Mapa de calor de ubicaciones de recogida y destino mas frecuentes (PULocationID, DOLocationID).
  • Analisis de tarifas: Promedio de fare_amount, tip_amount y total_amount por zona, tipo de pago y hora del dia.
  • KPI operativos: Distancia promedio por viaje, ingresos por dia, propinas por zona.

Conclusiones Batch: El pipeline automatizado Cloud Scheduler + Pub/Sub + Cloud Run + Cloud Storage + Dataprep/Alteryx + BigQuery + Power BI demostro ser eficaz para procesamiento masivo. Dataprep fue vital para erradicar tarifas negativas y duplicados. BigQuery habilito auditoria SQL. Power BI transformo datos crudos en conocimiento estrategico para optimizar operaciones de transporte.
EV3 — SUBASTA DE COMPONENTES
Dataset final: ~5,600 registros de transacciones de subasta, cada uno con 10 campos. Datos generados via webhook simulado.

Hallazgos clave del dashboard en Looker Studio:
  • Distribucion por genero: 63.2% femenino, 36.8% masculino, reflejando la composicion demografica de los participantes.
  • Volumen por empresa: Meta, Salesforce y Alphabet muestran la mayor frecuencia de transacciones.
  • Montos por producto: Productos de mayor valor economico concentran las transacciones mas significativas (distribucion de cola larga).
  • Forma de pago: Concentracion en un metodo predominante, sugiriendo estandarizacion.

Conclusiones Streaming: El pipeline de streaming captura, procesa y visualiza datos en tiempo real con latencia aceptable para monitoreo operacional. La arquitectura serverless (Cloud Run + Pub/Sub + Dataflow) escala automaticamente. Ventanas de 12 segundos ofrecen balance entre granularidad y volumen de archivos.

Recomendaciones: Implementar Cloud Scheduler para pruebas automatizadas, alertas en Cloud Monitoring, BigQuery como destino final para consultas SQL. Para produccion, considerar retencion extendida en Pub/Sub y particionamiento por fecha en Cloud Storage.
TECNOLOGIAS DEL PIPELINE
EV2 — BATCH
Ingesta
Cloud Scheduler (cada minuto) → Pub/Sub (TOPIC_taxi) → Cloud Run (yellow-trip-date-api) descarga y procesa Parquet desde repositorio oficial NYC TLC.
Procesamiento
Cloud Run + Python/Flask + pandas + pyarrow: procesa Parquet en lotes de 150,000 filas y transmite CSV a Cloud Storage via streaming para evitar OOM.
Almacenamiento
Cloud Storage (yellow-trip-eva2) como Data Lake. Dataprep/Alteryx para limpieza ETL. BigQuery (Dataprep.taxis_limpios) como Data Warehouse.
Visualizacion
Power BI Desktop conectado a BigQuery. Dashboards: distribucion horaria, mapa de calor, analisis de tarifas, KPI operativos.
EV3 — STREAMING
Ingesta
Cloud Run + Python/Flask: microservicio REST que recibe eventos via HTTP POST. Desplegado como contenedor serverless, escala a cero.
Mensajeria
Pub/Sub con persistencia de 7 dias. Desacopla webhook de Dataflow, garantizando entrega at-least-once.
Procesamiento
Dataflow + Apache Beam: pipeline autoescalado con ventanas fijas de 12 segundos. Lee de Pub/Sub via PubsubIO, escribe CSV particionados a Cloud Storage.
Visualizacion
Looker Studio: dashboard interactivo con 8 graficos. Metricas: distribucion por genero, volumen por empresa, montos por producto.
Infraestructura compartida: Gestion dual via GCP Console (configuracion inicial y monitoreo) + scripts PowerShell/gcloud CLI (despliegues repetibles). Stack comun: Cloud Run, Pub/Sub, Cloud Storage, Alteryx/Dataprep. Diferenciador: BigQuery + Power BI (EV2 Batch) vs Dataflow + Looker Studio (EV3 Streaming).
