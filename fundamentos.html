FUNDAMENTOS — 5V
Volumen: Sucesion continua de eventos individuales cuya acumulacion justifica el almacenamiento escalable en Cloud Storage.

Velocidad: Dimension que define el proyecto. Pipeline streaming con ventanas de 12 segundos, latencia de minutos en lugar de horas (contraste con Batch EV2).

Variedad: Datos ingresan como JSON via webhook y se materializan como CSV con campos de texto, temporales y numericos.

Veracidad: Diez campos estandarizados (id_cliente, cliente, genero, id_producto, producto, precio, cantidad, monto, forma_pago, fecreg). Limpieza via Alteryx/Dataprep.

Valor: Dashboard en Looker Studio con indicadores clave: distribucion por genero, volumen por empresa y montos por producto.

LATENCIA DEL PIPELINE
Tiempo de procesamiento crudo (Data Lag): ~37 segundos — velocidad de reaccion interna de Dataflow al detectar y procesar el mensaje en memoria RAM.

Latencia de disponibilidad final: ~12 minutos — tiempo hasta que el dato se guarda fisicamente como archivo CSV en Cloud Storage.
ANALISIS DE DATOS
Dataset final: ~5,600 registros de transacciones de subasta, cada uno con 10 campos. Los datos fueron generados via webhook simulado durante la evaluacion.

Hallazgos clave del dashboard:
  • Distribucion por genero: 63.2% femenino, 36.8% masculino, reflejando la composicion demografica de los participantes en la subasta.
  • Volumen por empresa: Meta, Salesforce y Alphabet muestran la mayor frecuencia de transacciones, consistente con su presencia en el mercado de componentes tecnologicos.
  • Montos por producto: Los productos de mayor valor economico concentran las transacciones mas significativas, con una distribucion de cola larga tipica de mercados de subasta.
  • Forma de pago: La mayoria de las transacciones se concentran en un metodo de pago predominante, lo que sugiere estandarizacion en el proceso de compra.

Conclusiones: El pipeline de streaming demuestra la capacidad de capturar, procesar y visualizar datos en tiempo real con una latencia aceptable para aplicaciones de monitoreo operacional. La arquitectura serverless de GCP (Cloud Run + Pub/Sub + Dataflow) permite escalar automaticamente segun la carga de eventos, eliminando la necesidad de gestionar infraestructura. Las ventanas de 12 segundos en Dataflow ofrecen un balance entre granularidad de datos y volumen de archivos generados.

Recomendaciones: Implementar Cloud Scheduler para pruebas automatizadas, configurar alertas de monitoreo en Cloud Monitoring, y evaluar BigQuery como destino final para habilitar consultas SQL sobre los datos en tiempo real. Para produccion, considerar retencion de mensajes extendida en Pub/Sub y particionamiento por fecha en Cloud Storage.
TECNOLOGIAS DEL PIPELINE
El pipeline de datos se sostiene sobre un stack tecnologico que abarca desde la ingesta hasta la visualizacion:

Ingesta (Cloud Run + Python/Flask): Microservicio REST que recibe eventos via HTTP POST. Desplegado como contenedor serverless, escala a cero cuando no hay trafico. Codigo en Python con framework Flask, manejando errores de API de Google y reintentos.

Mensajeria (Pub/Sub): Bus de eventos asincrono con persistencia de 7 dias. Desacopla la produccion del consumo, permitiendo que el webhook y Dataflow operen independientemente. Garantiza entrega at-least-once.

Procesamiento (Dataflow + Apache Beam): Pipeline en Java con ventanas fijas de 12 segundos. Autoescalado hasta 10 workers. Lee de Pub/Sub via PubsubIO y escribe CSV particionados a Cloud Storage. El pipeline se monitorea desde la UI de Dataflow con metricas de throughput y lag.

Almacenamiento (Cloud Storage): Data Lake con carpetas estructuradas (temp/, staging/, samples/output/). Politica TTL de 30 dias para control de costos. Los CSV generados son la fuente de datos para las herramientas de visualizacion.

Preparacion (Alteryx / Dataprep): ETL visual para limpiar y estandarizar datos. Alteryx ofrece nodos de transformacion avanzada; Dataprep es la alternativa nativa GCP integrada directamente con Cloud Storage.

Visualizacion (Looker Studio): Dashboard interactivo con 8 graficos conectado a Cloud Storage. Actualizacion cada 15 minutos. Las metricas clave incluyen distribucion por genero, volumen por empresa y montos por producto.

Infraestructura (GCP Console + PowerShell): Gestion dual via interfaz web (GCP Console) y automatizacion por script (PowerShell/gcloud CLI). La consola web es ideal para configuracion inicial y monitoreo; los scripts garantizan despliegues repetibles y documentados.
